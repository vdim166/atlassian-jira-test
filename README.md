### Jira Project Assistant — Документация по использованию

## Описание

# Jira Project Assistant — это Forge-приложение для Atlassian, которое помогает анализировать и исправлять проблемы в задачах проекта:

находить не назначенные задачи
выявлять проблемы с приоритетами и дедлайнами
автоматически распределять задачи между участниками команды
анализировать загрузку команды

### 1. Регистрация приложения

```bash
npm install -g @forge/cli
forge login
forge register
```

### 2. Сборка и деплой

```bash
npm install
npm run build:ui
forge deploy
forge install
```

### Разработка с tunnel

```bash
npm run build:ui
forge tunnel
```

Или через Docker:

```bash
docker compose --profile forge up forge-tunnel
```

# При редиплой обязательно:

```bash
npm run build:ui
forge deploy
```

# Как использовать Jira Project Assistant

## 1. Открытие приложения

Открой любой Jira-проект и перейди во вкладку:

> **Project Assistant**

Приложение загрузится автоматически и подтянет данные проекта.

## 2. Выбор проекта

Используй выпадающий список **Project** в верхней части интерфейса.

После выбора проекта приложение:

- загрузит задачи проекта
- загрузит участников команды
- пересчитает статистику проекта

## 3. Просмотр задач проекта

На дашборде отображается:

- общее количество задач
- задачи без исполнителя
- задачи с низким приоритетом и близким дедлайном

Все данные обновляются автоматически при смене проекта.

## 4. Автоматическое назначение задач

Нажми кнопку:

> **Auto-assign unassigned**

Появится окно подтверждения.

После подтверждения:

- все неназначенные задачи распределяются между участниками команды
- интерфейс обновляется сразу (оптимистичное обновление UI)
- изменения синхронизируются с Jira в фоне

Если операция завершится с ошибкой — изменения будут отменены.

## 5. Ручное назначение задач

В каждой задаче:

- нажми кнопку **Assign**
- выбери пользователя из списка
- подтверди назначение

Изменение применяется сразу в интерфейсе и затем отправляется в Jira.

## 6. Изменение приоритета задачи

Если задача имеет низкий приоритет и близкий дедлайн:

- открой окно **Priority Fix**
- выбери новый приоритет (Medium или High)
- подтверди изменение

## 7. Команда проекта

Во вкладке **Team** отображается:

- список участников проекта
- количество назначенных задач
- уровень активности (low / medium / high)

Активность рассчитывается на основе текущей нагрузки и недавних обновлений задач.

## Обработка ошибок

Если операция не удалась:

- интерфейс возвращается к предыдущему состоянию
- сообщение об ошибке показывается в верхней части экрана

## Система оценки активности команды

Активность каждого участника рассчитывается на основе назначенных задач и их недавних обновлений.

### Расчёт score

Для каждой задачи:

- +10 очков за назначенную задачу
- +15 очков, если задача обновлялась за последние 7 дней

### Пагинация

Пагинация работает по 10 задач на странице и может быть изменена в файле shared/constants.ts (ROWS_PER_PAGE)
